1. ゲインの仕組み（5段の掛け算）

各チャンネルの Total Gain は、5段のアンプ設定の掛け算で決まります。

Total Gain ＝ Gain1 × Gain2 × Gain3 × Gain4 × Gain5

Low ch（CH1〜8）の各段の役割

段入力方式取りうる値倍率の働き調整での役割
Gain12択-0.2 / -0.4×2 段粗調整
Gain22択-1.0 / -4.0×4 段粗調整
Gain3自由入力-0.5 〜 -1.0（連続）×0.5〜1.0微調整
Gain42択2.0 / 4.0×2 段粗調整
Gain5固定-1.35固定—

※ 符号は途中段で負ですが、負の段が4つあるため Total Gain は正になります（画面には %.2f倍 で表示）。

2. なぜ調整しにくいのか / 攻略の鍵

「5つの入力があり、1つは自由入力、残りは2択」という構成のため、闇雲に触ると組み合わせが多くて目標値に合わせにくくなります。しかし仕組みを分解すると、調整は実質2種類の操作に整理できます。

🔑 攻略の鍵

  • 2択スイッチ（Gain1 / Gain2 / Gain4） は、いずれも×2 単位でしか倍率を変えられません。つまり 0.27 → 0.54 → 1.08 → 2.16 → 4.32 → 8.64倍 … という飛び飛びの「帯（オクターブ）」を選ぶ操作です。
  • Gain3（自由入力） は連続値を取れる唯一の段。その可変幅は 0.5〜1.0倍＝ちょうど1オクターブ分です。

→ 2択で「どの帯に乗るか」を選び、Gain3 で「帯の中を連続スライド」して目標に合わせる。各帯はちょうど隣と接するため、0.27倍〜8.64倍まで隙間なく連続に設定できます。

つまりあなたの直感どおり、「自由入力（Gain3）で合わせ込む」が正解です。 2択は最初にだいたいの帯を決めるためだけに使い、最後の微調整は Gain3 一本に集約します。

3. 推奨手順（粗→微の2ステップ）

  1. 目標の Total Gain（倍）を決める。 設定可能範囲は 0.27倍 〜 8.64倍。
  2. 【粗調整】帯を選ぶ。 目標値以上で最も近い基準倍率（0.54 / 1.08 / 2.16 / 4.32 / 8.64）になるよう、下の早見表から Gain1 / Gain2 / Gain4 の2択を設定する。
  3. 【微調整】Gain3 を入れる。 Gain3 ＝ −(目標倍率 ÷ 基準倍率) を計算して自由入力欄へ。値は必ず -1.0 〜 -0.5 に収まります。
  4. 確認。 画面の「○○倍」表示が目標どおりか確認し、ズレていれば Gain3 だけを微修正する（2択は触らない）。

迷ったら下の計算機に目標倍率を入れれば、上の2〜3を自動で出します。

4. 逆算計算機（目標倍率 → 設定値）

Low ch: 0.27〜8.64倍 を連続設定可。High ch: 0.5 / 1.0 / 2.0 / 4.0倍 の4段階のみ（自由入力なし）。

5. 早見表（Low ch CH1〜8）

「基準倍率」は Gain3 = -1.0（最大）のときの Total Gain。Gain3 を -1.0→-0.5 に動かすと、その帯の中で 基準倍率 〜 基準倍率×0.5 まで連続に下がります。

基準倍率Gain1Gain2Gain4Gain5Gain3で作れる範囲
0.54倍-0.2-1.02.0-1.350.27 〜 0.54
1.08倍-0.2-1.04.0-1.350.54 〜 1.08
2.16倍-0.2-4.02.0-1.351.08 〜 2.16
4.32倍-0.2-4.04.0-1.352.16 〜 4.32
8.64倍-0.4-4.04.0-1.354.32 〜 8.64

※ 1.08 / 2.16 / 4.32 倍は別の2択組み合わせ（Gain1=-0.4 側）でも作れますが、表は代表的な1組を記載。どれを使っても Total は同じです。

6. High ch（CH9〜13）について

High ch は Gain3 / Gain4 / Gain5 が固定（自由入力なし）。可変なのは Gain1・Gain2 の2択だけなので、微調整はできず4段階のみです。これはハード構成上の仕様です。

Total GainGain1Gain2
0.5倍-0.5-1.0
1.0倍-0.5-2.0
2.0倍-2.0-1.0
4.0倍-2.0-2.0

※ CH13 のみ Gain3 = -1.0 のため Total の符号が反転しますが、倍率の大きさは上表と同じです。

7. 入力ルール・注意事項

  • Gain3 の入力範囲は -1.0 〜 -0.5。範囲外を入れると欄が赤くハイライトされ、内部では自動でクランプされます。
  • Gain3 はハードで 0.5 / 511 ≒ 0.00098 刻みに量子化されるため、入力値どおりピッタリにならず微小な丸めが出ることがあります。最終確認は画面の「○○倍」表示で行ってください。
  • 設定後は Gain トリガ書き込み（0x1 → GAIN_TRG）で初めてハードに反映されます。アプリの適用操作を忘れずに。
  • 微調整は Gain3 だけを動かすのが鉄則。2択を変えると帯ごと2倍ずれるので、合わせ込み中は触らないこと。
UI改善のヒント（開発者向け）

この計算機のロジックをアプリ本体に組み込み、「目標 Total Gain を入力 → Gain1/2/4 の2択と Gain3 を自動設定」ボタンを付けると、ユーザーは倍率を1つ入れるだけで済みます。さらに Gain3 をスライダー化すれば「帯の中を連続で動かす」操作が直感的になります。

このマニュアルは Dialog1_Main.cpp / FpgaRegisterLogic.h の実装値（strGainMultp, CalcGain3RegValue）に基づいて作成。
